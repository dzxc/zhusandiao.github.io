githug的简单介绍

这是一个通过游戏的形式将开发中使用 git 工具可能碰见的场景转化成一个个的小关卡，闯关者通过一个个关卡的同时，潜移默化地掌握了git知识。

闯过这 54 关，点亮你的 Git 技能树
Github地址

溯本追源–git

时间： 2005 年 4 月
本质：一个分布式 (distributed) 的性能优异的版本控制系统 (VCS) 工具

githug游戏

安装：githug 的安装在最上面给出的的两个相关链接里都有提到，就不重复了。

玩法：每一关的提交命令githug play ，实际上直接输入githug就可以了，查看所有关卡命令githug levels，提示命令githug hint ，关卡重置命令githug reset，跳转到任意关卡命令githug reset 关卡名称，例如：githug reset add，就是跳转到第三关 add

create githug directory
首先创建 githug 目录
yn–>y

1. init

在刚刚创建的 githug 目录下初始化一个 git 仓库
先切换到 githug 目录下cd git_hug，然后 git init

2. config

设置 name 和 email
git config --global user.name "drops"
git config --global user.email "wei..."

3. add

将一个叫 README 的文件，从工作区添加到暂存区 (stageing area) ,以便追踪
git add README

4. commit

提交commit信息
git commit -m "add README"

5. clone

从一个 github 地址克隆仓库
git clone https://github.com/Gazler/cloneme
注：如果仓库地址后面不加参数，则默认 clone 到当前目录，与后面一关对照

6. clone_to_folder

从 github 地址克隆仓库到指定文件夹下
git clone https://github.com/Gazler/cloneme my_cloned_repo

7. ignore

在git仓库中忽略指定文件，即不追踪这些文件
vi .gitignore，打开 vim 编辑 .gitignore 文件

添加这一行 * .swp

8. include

git仓库中停止追踪 .a 结尾的文件，但不包括 lib.a
vi .gitignore，打开 vim 编辑 .gitignore ** ![include](http://7fva1q.com1.z0.glb.clouddn.com/githug%238-include.png) 添加这两行 ** .a**，**!lib.a`*

9. status

查看仓库中未追踪的文件
git status，可以查看待提交的变更和未追踪的文件

10. number_files_commited

查看待提交的文件数目
仍然是 git status`` ，可以直接看到要提交的文件

11.rm

删除一个已经在工作区中删除，但还存在于仓库中的文件
先 git status ，查看仓库状态，发现有一个待删除的文件
git rm 待删除文件 ，删除仓库中的该文件，接着git commit ，将改动提交到仓库中

12.rm_cached

删除已经添加到暂存区的某一文件，但在文件系统中保留该文件
git rm --cached 待删除文件

13.stash

暂存工作状态，但是不提交
git stash

14. rename

修改仓库中某个文件的名称
git mv oldfile.txt newfile.txt，此时修改已经被暂存但还未提交

15. restructure

新建一个文件夹，并将原仓库中的某些文件移到这个新建的文件夹中
git mkdir src，git mv *.html src
其中 git mkdir 目录名 是新建目录， git mv 文件 目录 是移动文件命令

16. log

查看提交日志
git log

17. tag

为最近的一次 commit 打上标签
git tag 标签名

18. push_tags

把一些标签推送到远程仓库
git push tags

19. commit_amend

修改上一次的提交，加入一个新的文件
git add forgotten_file.rb ，先将文件存入暂存区，然后 git commit amend ，使用一次新的提交，替代上一次的提交

20. commit_in_future

为一次提交指定一个未来的时间
git commit --date=12.06.2016T15:00:00

21. reset

把某个文件撤出暂存区
git reset HEAD 要撤出的文件
git reset HEAD to_commit_second.rb
