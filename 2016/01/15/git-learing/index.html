githug的简单介绍

这是一个通过游戏的形式将开发中使用 git 工具可能碰见的场景转化成一个个的小关卡，闯关者通过一个个关卡的同时，潜移默化地掌握了git知识。

闯过这 54 关，点亮你的 Git 技能树
Github地址

溯本追源–git

时间： 2005 年 4 月
本质：一个分布式 (distributed) 的性能优异的版本控制系统 (VCS) 工具

githug游戏

安装：githug 的安装在最上面给出的的两个相关链接里都有提到，就不重复了。

玩法：每一关的提交命令githug play ，实际上直接输入githug就可以了，查看所有关卡命令githug levels，提示命令githug hint ，关卡重置命令githug reset，跳转到任意关卡命令githug reset 关卡名称，例如：githug reset add，就是跳转到第三关 add

create githug directory
首先创建 githug 目录
yn–>y

1. init

在刚刚创建的 githug 目录下初始化一个 git 仓库
先切换到 githug 目录下cd git_hug，然后 git init

2. config

设置 name 和 email
git config --global user.name "drops"
git config --global user.email "wei..."

3. add

将一个叫 README 的文件，从工作区添加到暂存区 (stageing area) ,以便追踪
git add README

4. commit

提交commit信息
git commit -m "add README"

5.

从一个 github 地址克隆仓库
git clone https://github.com/Gazler/cloneme
注：仓库地址后面不加参数，则默认 clone 到当前目录，与后面一关对照
