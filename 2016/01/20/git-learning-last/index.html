最后的36~54关

- 36. delete_branch

删除某个分支
git branch -d [branch_name]

- 37. push_branch

推送指定分支到远程仓库中
git push <远程主机名> <本地分支名>:<远程分支名>

该关卡中远程主机名为 origin 因为不想合并到主分支中，直接推送到 test_branch 中 ，
所以git push origin test_branch:test_branch

- 38. merge

合并指定分支内容到当前分支
git merge [branch_name]

- 39. fetch

获取远程仓库的修改，但是不合并到当前分支
git fetch [远程主机名]

- 40. rebase

这个命令，我还不怎么理解，本关卡中，有两个分支 master 和 feature 。
先查看 master 的提交记录

然后查看 feature 的提交记录

接着git rebase master feature
再查看 master 分支的提交记录

似乎是将 feature 分支的提交记录合并了

- 41. repack

将仓库中冗余的打包文件移除
git repack

查看Git 是如何存储对象的

- 42. cherry-pick

cherry-pick命令”复制”一个提交节点并在当前分支做一次完全一样的新提交
git checkout-pick [某个commit的hash值]

在本关卡中，先查看所有的提交记录 git log -all

记录 Filled in REASME.md with proper input 这次 commit 的 hash 值
接着 git checkout-pick ca3236

- 43. grep

grep 在 linux 中就是一个强大的搜索命令，支持文本、正则等方式
本关卡中git grep TODO

- 44. rename_commit

重命名某次提交
先git log，查看提交信息，记录有输入错误的 commit 的前一次 commit 的 hash 值
然后git rebase -i hash值
接着会出现两次默认编辑器窗口，第一次是修改 .git/rebase-merge/git-rebase-todo ，将出错的那行 pick 改成 reword ，同时修改后面的错误，保存之后
又会出来一个窗口，这次修改的是 .git/COMMIT_EDITMSG 手动修改提交信息中的错误，再保存就可以了
