通过githug游戏学习git（中）

发表于 2016-01-20   |   阅读次数

另外两篇

通过githug游戏学习git（上）
通过githug游戏学习git（下）

下面是16~35关

- 16. log

查看提交日志
git log

- 17. tag

为最近的一次 commit 打上标签
git tag 标签名

- 18. push_tags

把一些标签推送到远程仓库
git push tags

- 19. commit_amend

修改上一次的提交，加入一个新的文件
git add forgotten_file.rb ，先将文件存入暂存区，然后 git commit amend ，使用一次新的提交，替代上一次的提交

- 20. commit_in_future

为一次提交指定一个未来的时间
git commit --date=12.06.2016T15:00:00

- 21. reset

把某个文件撤出暂存区
git reset HEAD 要撤出的文件
git reset HEAD to_commit_second.rb

- 22. reset_soft

只撤销上次的提交信息，工作区和暂存区内容不变
git reset --soft HEAD^1

- 23. checkout_file

丢弃工作区某个文件的改动，用上次提交覆盖工作区
git checkout [commit] [file]
git checkou con*
注：图中的这个实际上是用暂存区的文件覆盖，实际上 checkout 还有几种，以及需要和 git reset 命令进行区分，这些留待以后说

- 24. remote

查看远程仓库
git remote

- 25. remote_url

查看远程仓库以及地址
git remote -v

- 26. pull

从（远程）仓库取回变动，并与本地分支合并
git pull [remote] [branch]
git pull origin master

- 27. remote_add

添加一个远程仓库，同时命名，git remote add [shortname] [url]
git remote add origin https://githug.com/githug/githug

- 28. push

推送本地修改到远程仓库中，这个例子里，远程的仓库和分支是 origin/master ，本地主机的分支是master，先 git rebase origin/master ，在当前分支上，合并 origin/master ，然后通过 git push origin master 推送本地修改到这个远程仓库

git push [remote] [branch]，上传本地指定分支到远程仓库

- 29. diff

显示暂存区和工作区的差异
git diff，注：diff信息读取也值得一讲，也留待以后细说

- 30. Blame

显示指定文件是什么人在什么时间修改过
git blame [file]

- 31. branch

创建分支
git branch [branch_name]

- 32. checkout

创建并切换分支
git branch my_branch 先创建一个名为 my_branch 的分支，接着 git checkout my_branch ，通过 git checkout 命令切换到该分支
实际上，开发中一般直接使用git checkout -b [branch_name]，直接创建分支并且切换过去

- 33. checkout_tag

切换到指定 tag
git checkout tag_name，可以先git tag 查看所有的tag ，发现有 v1.2，然后git checkout v1.2，切换过去

- 34. checkout_tags_over_branch

切换到指定 tag ，但这个 tag 和 branch 重名了
git checkout tags/tag_name，还是先git tag 查看所有的tag，接着git checkout tags/v1.2直接切换

- 35. branch_at

根据一个特定提交创建一个分支
该关卡的要求是在上一次的提交前创建新分支，git branch test_branch HEAD^1
