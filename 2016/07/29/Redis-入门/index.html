Redis 入门

发表于 2016-07-29   |   阅读次数

Redis的简单介绍

  • 特点

    • 是键值型的数据库
    • 支持数据的持久化，可以将内存中的数据保存在磁盘中。
    • 支持多种数据结构的存储
    • 支持数据的备份

  • 优势：

    • 性能极高：读110000次/s，写的速度是81000次/s
    • 操作是原子性的

Redis支持数据结构

Redis 支持 5 种数据类型：

  • string（字符串）
  • hash（哈希）
  • list（列表）
  • set（集合）
  • zset(sorted set：有序集合，redis1.2版本开始)
  • string
    • 是 redis 的最基本的类型，一个key对应一个value
  • hash
    • Redis hash 是一个键值对集合。
    • 一个string类型的field和value的映射表，hash特别适合用于存储对象。
  • list
    • 列表是简单的字符串列表，按照插入顺序排序。
    • 可以在列表的头部（左边）或者尾部（右边）添加一个元素。
  • set
    • Redis的Set是string类型的无序集合。
  • Redis zset
    • 和 set 一样也是string类型元素的集合,且不允许重复的成员
    • 每个元素都会关联一个double类型的分数
    • 通过分数来为集合中的成员进行从小到大的排序
    • zset的成员是唯一的,但分数(score)却可以重复

操作 sorted set 的命令

  • Redis sorted set 和 Set 一样的地方是：都是string 类型元素的集合，而且成员不允许重复
  • 不同的地方是：每个元素都会关联一个double类型的分数，Redis 通过分数来为集合中的成员进行从小到大的排序。

注：sorted set 的成员是唯一的,但分数(score)却可以重复。

示例，存储了一个 sorted set –NBAplayer。通过分数由小到大，表示一个球员的进步轨迹–由菜鸟到历史最佳。

HyperLogLog 结构

  • Redis HyperLogLog 是用来做基数统计的算法。
    优点：在输入元素的数量或者体积非常非常大时，计算基数所需的空间总是固定的、并且是很小的。–节省资源，性能强大。
  • HyperLogLog 只计算，不存储

基数的概念：

比如数据集 {1, 3, 5, 7, 5, 7, 8}， 那么这个数据集的基数集为 {1, 3, 5 ,7, 8}, 基数(不重复元素)为5。
即描述一个数据集所有元素的最小集合。

基数估计

基数估计就是在误差可接受的范围内，快速计算基数。

示例：在一个工厂里有多个职位，如经理，工人，保安和老板等，每个职位都可能有多个员工，如果我只是想了解这个工厂有多少个职位而不关心每个岗位上有多少个员工，我可以使用基数估计。

可以看到，一共有四个岗位。

Redis 发布订阅

Redis 发布订阅(pub/sub)是一种消息通信模式：

发送者(pub)发送消息，订阅者(sub)接收消息。

Redis 客户端可以订阅任意数量的频道。

Redis 事务

Redis 事务可以一次执行多个命令。并且保证以下两点：

  • 1.)事务是一个单独的隔离操作：
    • 事务中的所有命令都会序列化、按顺序地执行。事务在执行的过程中，不会被其他客户端发送来的命令请求所打断。
  • 2.)事务是一个原子操作：
    • 事务中的命令要么全部被执行，要么全部都不执行。
  • 3.）一个事务从开始到执行会经历以下三个阶段：
    1. 开始事务
    2. 命令入队
    3. 执行事务

示例：
先用 multi 开始一个事务，然后将多个命令入队到事务中，最后由 EXEC 命令触发事务，同时执行所有命令。

下面是Redis事务相关指令

序号 命令及描述
1 discard 取消事务，放弃执行事务块内的所有命令。
2 exec 执行所有事务块内的命令。
3 multi 标记一个事务块的开始。
4 unwatch 取消 WATCH 命令对所有 key 的监视。
5 watch key [key …] 监视一个(或多个) key ，如果在事务执行之前这个(或这些) key 被其他命令所改动，那么事务将被打断。
